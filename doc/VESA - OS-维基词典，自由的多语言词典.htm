功能VESA †的

  • VESA信息采集
    • AX = 0x4f00;
    • ES：DI =支持（至少256字节）地址来接收信息;
    • 返回：
    • 如果该值是不VESA支持0x4f，否则，如果有支持的AL == VESA
    • 成功完成的功能，否则在函数退出时的错误，如果AH = 0
    • 存储的信息中的DI：ES
      抵销长度内容
      +0 X00DW签署“VESA”
      +0 X04瓦特版本的VESA（例：0x0102如果1.02）
      +0 X06段：ofs16一个到名称字符串OEM（ASCIZ的的指针）
      +0 X0ADW各种标记
      +0 X0E段：ofs16指针到一个数组中的视频模式的枚举，数组字，为0xFFFF终止
      +0 X12瓦特VRAM容量，支持VESA（KB）
      +0 X14242bytes储备
  • 信息采集屏幕模式
    • AX = 0x4f01;
    • 获得的信息屏幕模式的号码CX =;
    • ES：地址来接收信息屏幕模式DI =（256字节以上）;
    • 返回：
    • 如果该值是不VESA支持0x4f，否则，如果有支持的AL == VESA
    • 成功完成的功能，否则在函数退出时的错误，如果AH = 0
    • 存储的信息中的DI：ES
      抵销长度内容
      +0 X00瓦特Mode属性
      +0 X02b一个窗口属性
      +0 X03b窗口的属性乙
      +0 X04瓦特寻址的窗口单位（KB）
      +0 X06瓦特窗口大小（以KB为单位）
      +0 X08瓦特窗口的起始地址A（段落）
      +0 X0A瓦特窗口B的起始地址（用于段落）
      +0 X0C段：ofs16IP：CS改变常规的实模式窗口
      +0 X10瓦特在VRAM上的一条线中的字节数
      +0 X12瓦特分辨率的X
      +0 X14瓦特Y轴分辨率
      +0 X16b在X方向上的一个字符的点数
      +0 X17b在Y方向上的一个字符的点数
      +0 X18b
      +0 X19b一个像素或多个位（在短的颜色数）
      +0 X1Ab
      +0 X1Bb内存模型（0x04的= 0X06 = 256调色板，直接颜色）
      +0 X1Cb
      +0 X1Db
      +0 X1Eb
      +0 X1Fb
      +0 X20b
      +0 X21b
      +0 X22b
      +0 X23b
      +0 X24b
      +0 X25b
      +0 X26b
      +0 X27b
      +0 X28DW的VRAM基地地址
      +0 X2CDW屏幕VRAM的开始偏移量（对于那些使用免费的屏幕外的VRAM）
      +0 X30瓦特关闭屏幕VRAM大小（以KB为单位）
      +0 X32206bytes储备
  • 模式的设定画面
    • AX = 0x4f02;
    • 屏幕模式BX =;
      • 视频模式：BIT0-13
      • 然而，由于VESA 2.0 - 线性请求访问模式（要求1）：位14
      • （如图1所示）清晰删节VRAM：BIT15
    • 返回：
    • 如果该值为0x4f，否则不符合规定的，如果我有此功能对应的AL ==
    • 成功完成的功能，否则在函数退出时的错误，如果AH = 0
